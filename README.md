# Descrição do Trabalho

Este repositório apresenta os artefatos produzidos durante a simulação de uma Sprint Scrum proposta na disciplina de Engenharia de Software I.

A atividade teve como objetivo aplicar os principais conceitos do framework Scrum por meio da elaboração dos seus rituais e documentos, permitindo que a equipe praticasse planejamento, organização, acompanhamento e avaliação de uma Sprint.

---

# Objetivo da Sprint

Planejar, organizar e documentar uma Sprint Scrum para o desenvolvimento de um Sistema de Gestão de Projetos de Pesquisa.

O cenário proposto considera um chefe de grupo de pesquisa que deseja:

* Cadastrar projetos de pesquisa.
* Gerenciar atividades dos membros da equipe.
* Acompanhar o andamento dos projetos.
* Visualizar o progresso geral por meio de indicadores.

---

# Histórias de Usuário

### HU01

Como chefe, quero cadastrar um projeto com nome, descrição e prazo para organizar os projetos de pesquisa.

### HU02

Como chefe, quero adicionar atividades com título, responsável e status para acompanhar o andamento das tarefas.

### HU03

Como chefe, quero visualizar um dashboard com projetos e percentual concluído para acompanhar o progresso geral.

# Como Utilizar

1. Leia este README para compreender o contexto geral do trabalho.

2. Consulte o arquivo `SPRINT_BACKLOG.md` para visualizar as histórias de usuário, prioridades, tarefas e estimativas definidas durante o Sprint Planning.

3. Consulte o arquivo `DAILY_LOGS.md` para acompanhar os registros das Daily Scrums realizadas durante a Sprint.

4. Consulte o arquivo `RETROSPECTIVE.md` para visualizar os resultados da Sprint Retrospective, incluindo lições aprendidas e ações de melhoria.
